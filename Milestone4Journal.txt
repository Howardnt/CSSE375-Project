April 8, 2026
spent evening with chapters 6, 22, and 25. Big shift in my head: refactoring for testing is mainly about breaking a dependency you can't fake without it. 
walked through LinterGuiFrame.onRun() looking for seams. ASM project conversion is built inside SwingWorker. If I want to test run pipeline without a real disk, I need a way to swap that out. Asked AI where a seam could go so analysis pipeline can be tested without running Swing. 
suggested Parameterize Constructor on a new service class. Clicked instantly. I'd been thinking about subclass-and-override, which would have worked but is more invasive. 

April 10, 2026 
the "sensing vs separation" idea keeps slipping out of my head, so I wrote it down. For LinterGuiFrame.onRun(), what I actually need is separation. Parameterize Constructor is the lightest version, subclass and override is heavier but more flexible and works without changing the production code at all if the class is non-final. Decided I'll do both on the new service class so I can demonstrate two Ch. 25 techniques from  same place.

April 12, 2026 sketched LinterService design
a LinterService with a run(Request) → Response shape, where Request bundles the project path plus the lists of checks to run, and Response holds the result and the formatted output text. Java records for both. Two constructors on the service: one that wires up the real ASM converter, and one that takes a function so tests can swap it
leaving the class non-final so tests can also subclass and override if they want. Two seams from the same class.

April 13, 2026 
tried sketching LinterService as an interface plus implementation pair. Got 30 minutes in and realized the extra interface wasn't buying anything.  there's only one real implementation, and tests don't need to swap it; they need to swap the project loader inside it. Backed out.
